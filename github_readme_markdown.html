Features

Core Functionality

  • Onboarding Flow - 4-slide introduction with service explanation
  • Group Timeline - Social feed with posts, likes, comments, and gratitude stamps
  • Post Detail - Full post view with comments and reactions
  • My Progress - Personal goal tracking with charts and badges
  • Weekly Report - Summary of highlights and support statistics

Health Features

  • Health Checkup Upload - Photo/PDF/manual data entry with OCR demo
  • Health Results - Easy-to-understand health checkup results
  • AI Video Explanation - AI avatar explains health data in simple terms

Social Features

  • Group Creation - Family/friends/symptom-based groups
  • Gratitude Stamps - Express thanks and appreciation
  • Mutual Support Recognition - Highlight helping behaviors
  • Goal Setting - Collaborative health goals

Getting Started

Prerequisites

  • Modern web browser (Chrome, Safari, Firefox, Edge)
  • For mobile testing: Local server or deploy to web

Installation

  1. Clone the repository
    git clone https://github.com/YOUR_USERNAME/family-health-meeting-system.git cd family-health-meeting-system
  2. Open in browser
    • Option 1: Double-click index.html or 08_welcome.html
    • Option 2: Use a local server
      # Python 3 python -m http.server 8000 # Python 2 python -m SimpleHTTPServer 8000 # Node.js (with npx) npx http-server
    • Option 3: Use VS Code Live Server extension
  3. View on mobile device
    • Connect to same WiFi network
    • Open http://YOUR_LOCAL_IP:8000/08_welcome.html
    • Or deploy to GitHub Pages / Netlify / Vercel

Design Specifications

Visual Design

  • Width: 375px (iPhone standard)
  • Primary Color: #4A90E2 (Gentle Blue)
  • Accent Color: #50C878 (Soft Green)
  • Font: System fonts (-apple-system, BlinkMacSystemFont)
  • Tone: Warm, supportive, encouraging

Technical Stack

  • Pure HTML5/CSS3/JavaScript
  • No external dependencies
  • No backend required
  • Fully self-contained
  • Mobile-first responsive design

Important Notes

This is a Prototype

✅ What Works

  • Screen transitions work like a real app
  • Interactive elements are functional
  • All files use English filenames

❌ Limitations

  • No data persistence (reloading resets state)
  • No backend/API integration
  • OCR and AI features are demos only
